# [기능명세서] 모아성 1차 MVP

## 1. 핵심 기능 정의

<aside>

#### 파트너 초대 온보딩

<aside>

- **기능 설명**
    - 가입 직후 초대 코드를 통해 파트너를 연결하고, 카테고리 기반 선택만으로 이번 주 할 일을 등록해 서비스를 시작하는 기능
- **핵심 가설**
    - 가입 직후 파트너를 초대하는 흐름을 제공하면, 초대 수락뿐 아니라 양쪽 모두의 첫 할 일 등록까지 이어지는 활성화율이 높아질 것이다.
- **검증 포인트**
    - 통계 고도화나 게임화 요소보다 먼저, 유저가 '혼자 쓰는 앱'이 아닌 '함께 쓰는 공간'으로 진입해 실제 등록까지 완주하는지부터 검증해야 한다.
- **검증 지표**
    - `파트너 초대 성공률`, `주간 할 일 등록·완료율`과 직접 연결된다.
</aside>

#### **가사 리스트 및 완수율 시각화**

<aside>

- **기능 설명**
    - 등록된 할 일의 완료·미완료 여부를 수행자별로 구분 표시하고, 전체 완수율을 성(城)이 채워지는 이미지로 보여주는 기능
- **핵심 가설**
    - 완료·미완료 여부와 수행자를 한눈에 보여주면, 그동안 서로에게 보이지 않던 기여가 드러나 상대의 가사 수행에 대한 인지도가 실제로 향상될 것이다.
- **검증 포인트**
    - 통계 대시보드 고도화나 데이터 내보내기보다 먼저, 완료 여부와 수행자 표시만으로 상대의 기여를 인지시키는 최소 구조가 작동하는지 검증해야 한다.
- **검증 지표**
    - `주간 할 일 등록·완료율`, `4주 리텐션`, 그리고 설문 지표인 `인지도 향상 점수`와 직접 연결된다.
</aside>

#### **인정 루프 (리액션·편지·주기 종료 칭찬)**

<aside>

- **기능 설명**
    - 완료된 항목에 이모지 리액션이나 편지로 즉시 반응할 수 있고, 매주 주기 종료 시점에 칭찬 작성을 유도하는 화면을 노출하는 기능
- **핵심 가설**
    - 가사를 함께 등록하고 완료 여부를 확인할 수 있는 공유 구조를 제공하면, 서로의 노동을 더 잘 인지하게 되어 인정으로 이어질 것이다.
- **검증 포인트**
    - 따라서 MVP에서는 게임화 요소(포인트·배지·순위)보다 먼저, 완료가 실제로 리액션·편지 같은 인정 행동으로 전환되는지부터 검증해야 한다.
- **검증 지표**
    - `주간 완료+인정 완주 커플 수`, `인정 전송률`과 직접 연결된다.
</aside>

#### **편지 탭 (리액션·편지 히스토리 조회)**

<aside>

- **기능 설명**
    - 그동안 주고받은 리액션과 편지를 시간순으로 모아볼 수 있는 기능
- **핵심 가설**
    - 완료와 인정이 축적되는 구조를 제공하면, 일회성 반응에 그치지 않고 반복 사용(리텐션)으로 이어질 것이다.
- **검증 포인트**
    - 정보 비교·큐레이션 같은 부가 기능보다 먼저, 쌓인 인정 기록을 다시 꺼내보는 행위 자체가 재방문 동기로 작동하는지 검증해야 한다.
- **검증 지표**
    - `4주 리텐션`, 보조 지표인 `홈 탭 주간 재방문율`과 직접 연결된다.
</aside>

</aside>

<aside>

- **회원가입 (신규 유저)**
    1. 사용자는 A-01 시작화면에서 `새로 시작하기` 버튼을 클릭한다.
    2. 사용자는 A-02 화면에서 닉네임(10자 이내)을 입력하고 이모지 15개 중 하나를 선택한다.
    3. 사용자는 `시작하기` 버튼을 클릭하면 시스템이 순서대로 유효성을 검사한다.
        - 닉네임 미입력 → "닉네임을 입력해 주세요." Alert 노출
        - 이용약관 미동의 → "이용약관에 동의해 주세요." Alert 노출
        - 모든 조건 충족 → A-03 간편 로그인 연동 화면으로 이동한다.
    4. 사용자는 A-03 화면에서 카카오 또는 구글 버튼을 탭해 간편 로그인을 진행한다.
        - 탭 시 로딩 스피너가 표시되고 버튼이 비활성화되며, 소셜 로그인 인증 팝업이 호출된다.
        - 신규 유저 인증 완료 → A-04 초대 코드 화면으로 이동한다.
        - 기존 유저로 확인되면 "이미 존재하는 계정이에요. 로그인할까요?" 컨펌 팝업이 노출된다.
            - '네' 선택 시 → C-01 메인화면으로 이동한다.
            - '아니오' 선택 시 → A-03 화면을 유지한다.
        - 인증 실패 시 → "로그인에 실패했어요. 다시 시도해 주세요." Alert가 A-03 화면 위에 노출된다.
        - 네트워크 오류 시 → "인터넷 연결을 확인해 주세요." Alert가 A-03 화면 위에 노출된다.
    5. 사용자는 A-04 화면에서 파트너 초대 코드를 입력하거나 내 초대 코드를 생성한 뒤 `다음으로` 버튼을 클릭한다.
        - 초대 코드 생성 시, 모달에 코드와 복사 버튼이 노출되며 코드는 1회 생성 후 고정된다. 모달을 닫으면 버튼 텍스트가 "초대 코드 다시 보기"로 바뀐다.
    6. 시스템은 `다음으로` 버튼 탭 시 파트너 코드 입력 여부에 따라 유효성을 검사한다.
        - **파트너 코드를 입력한 경우**
            - 유효한 코드 + 파트너 할 일 목록 있음 → C-01 메인화면으로 바로 이동한다.
            - 유효한 코드 + 파트너 할 일 목록 없음 → "파트너가 아직 할 일을 등록하지 않았어요. 내가 먼저 등록해볼까요?" 컨펌 팝업이 노출된다.
                - '네' 선택 시 → A-06 할 일 선택 화면으로 이동한다.
                - '아니오' 선택 시 → 팝업만 닫히고 A-04 화면을 유지한다.
            - 잘못된 코드 → "올바르지 않은 초대 코드예요." Alert가 노출된다.
            - 이미 사용된 코드 → "이미 연결된 코드예요." Alert가 노출된다.
            - 네트워크 오류 → "인터넷 연결을 확인해 주세요." Alert가 노출된다.
        - **파트너 코드를 입력하지 않은 경우**
            - 내 초대 코드를 생성한 상태 → A-06 화면으로 이동한다. (파트너 연결 대기 상태)
            - 내 초대 코드도 생성하지 않은 상태 → "아직 코드를 생성하지 않았어요. 연결을 나중에 할까요?" 컨펌 팝업이 노출된다.
                - '연결할게요' 선택 시 → 팝업만 닫히고 A-04 화면을 유지한다.
                - '나중에 할게요' 선택 시 → A-06 화면으로 이동한다. (파트너 미연결 상태)
    7. 사용자는 A-04 화면 하단의 `나중에 연결할게요` 텍스트 버튼을 탭해 바로 건너뛸 수도 있다.
        - 탭 시 "파트너와 연결을 나중에 할까요? 파트너와 함께해야 집이 완성돼요. 마이페이지에서 언제든 연결할 수 있어요." 컨펌 팝업이 노출된다.
            - '연결할게요' 선택 시 → 팝업만 닫히고 A-04 화면을 유지한다.
            - '나중에 할게요' 선택 시 → A-06 화면으로 이동한다. (파트너 미연결 상태)
    8. 사용자는 A-06 화면에서 할 일을 선택한 뒤 C-01 메인화면으로 이동한다.
</aside>

<aside>

- **로그인 (기존 유저)**
    1. 사용자는 A-01 시작화면에서 `로그인` 버튼을 클릭한다.
    2. 사용자는 A-05 간편 로그인 화면에서 로그인을 진행한다.
    3. 시스템은 가입 여부와 할 일 목록 존재 여부를 확인한다.
        - 기존 회원 + 할일O → C-01 메인화면으로 이동한다.
        - 기존 회원 + 할일X → A-06 할 일 선택 후 메인화면으로 이동한다.
        - 미가입 상태면 "회원가입 할까요?" 컨펌 팝업이 노출된다.
            - '가입할게요' 선택 시 A-02로 이동해 ①번 회원가입 경로를 따른다.
            - '아니요' 선택 시 A-05 화면을 유지한다.
</aside>

#### 주기 전환 흐름도

![image.png](%5B%EA%B8%B0%EB%8A%A5%EB%AA%85%EC%84%B8%EC%84%9C%5D%20%EB%AA%A8%EC%95%84%EC%84%B1%201%EC%B0%A8%20MVP/image%202.png)

<aside>

- **공통**
    1. 사용자는 A-06 할 일 선택 화면에서 이번 주 할 일을 선택한다.
    2. 선택이 완료되면 C-01 메인화면으로 이동한다.
</aside>

<aside>

- **할 일 선택: 1주차 (최초)**
    1. 사용자는 할 일 선택 화면에서 전체 해제 상태의 항목 목록을 확인한다.
    2. 사용자는 카테고리 10개 중 원하는 항목을 직접 선택한다.
    3. 사용자는 선택한 항목별로 담당자를 지정한다.
        - 옵션: 나, 파트너, 미지정
    4. 담당자 지정이 완료되면 메인화면으로 이동한다.
- **할 일 선택: 2주차 이후**
    1. 사용자는 안내 박스를 통해 ‘이번 주 할 일을 이미 골라놨어요’ 메시지와 함께 지난주 선택 항목이 자동 체크된 상태를 확인한다.
        - `전체 해제` 버튼을 누르면 처음부터 다시 선택할 수 있다.
    2. 사용자는 필요 시 항목을 수정한 뒤 `선택 완료` 버튼을 클릭한다.
    3. 선택이 완료되면 메인화면으로 이동한다.
</aside>

<aside>

- **메인화면 (홈)**
    1. 사용자는 C-01 메인화면(홈)에서 할 일을 완료 체크한다.
    2. 완료 처리되면 할일 완수율이 업데이트된다.
    3. 시스템은 파트너에게 푸시 알림을 발송한다.
    4. 파트너는 알림을 확인하고 **C-02** 화면에서 리액션 또는 편지로 즉시 인정을 표현한다.
        - `보내기` 탭 시 → **C-04 편지 전송 완료** 화면으로 이동한다.
    5. 사용자는 메인화면 하단의 "이번 주 마무리" 버튼을 눌러 **B-01 주기 마감 모달**로 이동한다.
</aside>

<aside>

- **주기 전환 (매주 일→월)**
    1. 사용자는 **B-01 주기 마감 모달**에서 '이번 주 마무리하고 다음 주로 넘어가기'를 확인한다.
    2. **B-02 편지 작성** 화면에서 이번 주 현황(기여도: 할 일 완료 80% + 편지 작성 10% = 100%)을 확인하고, 편지를 작성한 뒤 `전송하고 다음 주 시작하기` 버튼을 클릭한다.
        - 편지 작성은 필수이며, 미작성 시 다음 단계로 진행할 수 없다.
    3. 편지 전송이 완료되면 시스템은 자동으로 **B-03 주간 통계** 화면으로 진입시킨다.
        - 둘 다 전송 + 완료 → 성 완공률 100%, 불꽃/벚꽃 연출과 함께 "이번 주 완성됨" 상태(주간 통계 2)로 표시되며, 조건 충족 시 연속 달성 배너가 함께 노출된다.
        - 한 명만 전송했거나 완수율이 낮음 → 미완성 상태(주간 통계 1)로 표시된다.
    4. 사용자는 B-03 화면에서 기여도 그래프·주간 통계 카드(완료 항목 수, 내 기여도, 파트너 기여도)를 확인한다.
    5. 사용자는 하단 `다음 할 일 설정하러 가기` 버튼을 눌러 **A-06 할 일 선택** 화면(2주차 이후 플로우, 지난 주 항목 자동 불러오기)으로 이동한다.
</aside>
